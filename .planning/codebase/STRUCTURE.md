## Directory Purposes

**`src/`:**
- Purpose: All TypeScript source files — the only directory developers edit
- Contains: One file per logical unit (background, popup, content scripts)
- Key files: `src/background.ts`, `src/popup/popup.ts`, `src/content/all-fields.ts`

**`src/popup/`:**
- Purpose: Extension action popup — UI and dispatch logic
- Contains: `popup.ts` (logic), `popup.html` (markup), `popup.css` (styles)
- Key files: `src/popup/popup.ts` is the only JS logic; it imports nothing (pure Chrome API calls)

**`src/content/`:**
- Purpose: Content scripts injected into CRM pages on demand
- Contains: One `.ts` per tool, plus a companion `.css` per tool that needs a panel
- Key files: `src/content/all-fields.ts` (main feature), `src/content/hello-world.ts` (demo)

**`dist/`:**
- Purpose: Built extension — load as an unpacked Chrome extension from this directory
- Generated: Yes — do not edit directly
- Committed: No — generated by `npm run build`
- Mirrors `src/` structure with `.ts` → `.js` compilation

**`icons/`:**
- Purpose: Extension icon assets at all required Chrome sizes
- Contains: `icon16.png`, `icon32.png`, `icon48.png`, `icon128.png`
- Committed: Yes — source assets
- Copied to `dist/icons/` on every build

## Key File Locations

**Entry Points:**
- `src/background.ts`: Service worker — currently a placeholder comment
- `src/popup/popup.ts`: Popup script — button listeners + `chrome.scripting.executeScript` calls
- `src/popup/popup.html`: Popup markup — loaded when user clicks the extension icon
- `src/content/all-fields.ts`: All Fields content script — full CRM field inspector
- `src/content/hello-world.ts`: Hello World content script — smoke test alert

**Configuration:**
- `manifest.json`: Chrome Extension MV3 manifest — declares service worker, popup, permissions
- `tsconfig.json`: TypeScript compiler options — strict mode, `noEmit: true`, types `["chrome", "xrm"]`
- `eslint.config.js`: ESLint flat config
- `build.js`: esbuild config + build orchestration (lint, copy statics, optional watch mode)
- `package.json`: npm scripts — `build`, `dev`, `watch`, `typecheck`, `lint`, `check`

**Core Logic:**
- `src/content/all-fields.ts`: Contains `main()`, `buildPanel()`, `formatValue()`, `injectStyles()`
- `src/popup/popup.ts`: All tool dispatch logic

**Styles:**
- `src/popup/popup.css`: Popup panel UI styles (260px wide, Segoe UI, blue `#1e64c8` brand)
- `src/content/all-fields.css`: Side-panel styles for the All Fields overlay (also duplicated inline in `all-fields.ts`)

## Naming Conventions

**Files:**
- Kebab-case for all files: `all-fields.ts`, `hello-world.ts`, `popup.ts`
- Content scripts: named after the tool they implement (`all-fields`, `hello-world`)
- Each tool has matching `.ts` and optionally `.css` with the same base name

**Directories:**
- Lowercase, single-word or hyphenated: `popup/`, `content/`

**CSS Classes:**
- Popup: semantic BEM-lite names — `.tool-btn`, `.tool-group`, `.header-title`
- Content panel: prefixed with `cfp-` (CRM Fields Panel) — `.cfp-header`, `.cfp-body`, `.cfp-close`, `.cfp-null`, `.cfp-type`

**DOM IDs:**
- Panel root: `crm-tools-fields-panel` (constant `PANEL_ID`)
- Injected style tag: `crm-tools-fields-style` (constant `STYLE_ID`)

**TypeScript:**
- Functions: camelCase — `main()`, `buildPanel()`, `formatValue()`, `injectStyles()`
- Constants: `SCREAMING_SNAKE_CASE` — `PANEL_ID`, `STYLE_ID`, `LOG`

## Where to Add New Code

**New Tool (content script):**
1. Create `src/content/<tool-name>.ts` — implement with a `main()` function called at the bottom
2. Optionally create `src/content/<tool-name>.css` for panel styles
3. Add entry point to `build.js` `entryPoints`: `'content/<tool-name>': 'src/content/<tool-name>.ts'`
4. Add `copyFileSync` for any companion CSS in `copyStatics()` in `build.js`
5. Add a button to `src/popup/popup.html` with id `btn-<tool-name>`
6. Add a `click` listener in `src/popup/popup.ts` that calls `chrome.scripting.executeScript` with `files: ['content/<tool-name>.js']`

**New Popup UI Element:**
- Markup: `src/popup/popup.html`
- Styles: `src/popup/popup.css`
- Logic: `src/popup/popup.ts`

**Shared Utilities:**
- No shared module infrastructure exists yet
- Add as `src/lib/<utility-name>.ts` and import into content scripts (esbuild will bundle inline)

## Special Directories

**`dist/`:**
- Purpose: Unpacked Chrome extension — point `chrome://extensions` load at this directory
- Generated: Yes — by `npm run build` or `npm run dev`
- Committed: No
- Do not edit: regenerated on every build

**`node_modules/`:**
- Generated: Yes
- Committed: No
- Dev-only: All dependencies are `devDependencies` (no runtime npm packages)
